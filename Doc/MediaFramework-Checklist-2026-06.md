## Phase P0 — Critical correctness & stability

- [x] **P0-1 FFmpeg demux stop race + interrupt callback.** **Closed in Batch 12.** `MediaContainerSharedDemux` now installs an `AVIOInterruptCB` on `_fmt` (FFmpeg.AutoGen 8 wrapper verified) driven by `_demuxerStopRequest` / `_disposed`, treats `AVERROR_EXIT` during stop as clean, and refuses seek/restart if the demux thread still does not exit. The unsafe old path that cleared `_demuxerThread` / `_demuxerStopRequest` and then reused/freed native demux state after a timed-out join is gone; a stuck thread leaves native state deliberately leaked rather than use-after-freed.
- [x] **P0-2 VideoPlayer post-submit double-dispose (3 sites).** **Closed via** capturing the PTS before `Submit`, gating counters/notification on a `submitted` flag, and routing the event through a new `RaisePresented` helper that wraps the invocation in its own try/catch (never disposes the frame, never lets a subscriber exception reach the clock-driver thread) — applied in `OnVideoTick`, `PresentLatestQueuedFrame`, `TrySubmitHeldFrame`. Note: `VideoFrame.Dispose` is idempotent (`Interlocked.Exchange`), so the real symptom was a *premature* release of a frame an async output still held queued, not a literal double-free. Regression test `VideoPlayerTests.Submit_Success_With_Throwing_PresentationEvent_Does_Not_Release_Frame`. **Verification correction (2026-06-02):** the double-release was fixed in all three sites, but `PresentLatestQueuedFrame` (the `LatestOnTick` path, called from `OnVideoTick` on the clock-driver thread) was still doing a *direct* `FramePresentationTimePresented?.Invoke` rather than going through `RaisePresented` — so a throwing subscriber could still escape to the clock driver there. Now routed through `RaisePresented` like the other two sites.
- [x] **P0-3 VideoRouter.RemoveOutput leaks owned outputs.** **Closed via** capturing the registration under `_gate`, then disposing it (when `DisposeOutputOnRouterDispose`) *outside* the lock — `VideoOutputPump.Dispose` joins its drainer thread, so holding `_gate` during teardown is unsafe.
- [x] **P0-4 AudioRouter drains unrouted/unstarted sources.** **Closed via** gating the run-loop read on routing: only sources targeted by a route this chunk are read (reused per-chunk `HashSet` so a source feeding many routes is read once; muted-but-routed sources still advance — mixer mute semantics). `keepRunning` only auto-stops once routes exist AND every routed source is exhausted, so dynamic "route-last" graphs aren't killed and an empty/route-less router idles instead of stopping. MediaPlayer now routes its audio source to a `DiscardingAudioOutput` so "play to completion" / master-clock advance still works with no real output attached (a later clocked PortAudio output still becomes the pacing primary). Regression test `AudioRouterSourceGatingTests.RunLoop_UnroutedSource_IsNotConsumedUntilRouted`.
- [x] **P0-5a AudioRouter background rethrow → fault.** **Closed via** a `Faulted` event + `Fault` property (new `AudioRouterFaultedEventArgs`); `RunLoop`'s catch now records the fault and raises the event instead of rethrowing (the `finally` still tears down thread/cts); `Start` clears the prior fault. Regression test `AudioRouterFaultTests.RunLoop_SourceThrows_RouterFaultsInsteadOfCrashing`.
- [x] **P0-5b FFmpeg demux thread exception boundary.** **Closed via** wrapping `DemuxerThreadProc` in a catch that records `_demuxFault` (exposed as `DemuxFault`), signals EOF (`_fileReadCompleted = true`) and pulses `_queueGate` so blocked audio/video tracks wake and exhaust gracefully; `StartDemuxerThread` clears the fault.
- [x] **P0-5c NDI receiver fault boundaries.** **Closed via** a `_faultEx` field + `Fault` property + `Faulted` event on both receivers; `IsExhausted` now terminal on fault; `CaptureLoop` wrapped in try/catch (video uses a `CaptureLoopCore` split to avoid re-indenting) that records the fault, wakes the blocked video reader, and stops the router pulling audio. **Verification correction (2026-06-02 late pass):** the standalone `NDIVideoReceiver`/`NDIAudioReceiver` were fixed, but the combined public `NDISource` path used by HaPlay still had an unguarded capture loop. `NDISource` now has the same terminal `Fault`/`Faulted` boundary, wakes blocked video reads on fault, reports audio/video adapters exhausted, and sets connection state outside the logging branch.
- [x] **P0-6 Static/text/image frame lifetime.** **Closed.** `TextLayerSource` fixed via a refcounted generation buffer (`FrameBuffer : IDisposable`): each rasterise produces an immutable pooled buffer; emitted frames share it and hold a ref (no per-frame copy while static); a property change rasterises a new generation, and an old one returns to the pool only when the source and all referencing frames are disposed. Regression tests `TextLayerSourceTests.HeldFrame_NotMutated_ByLaterTextChange` / `HeldFrame_RemainsReadable_AfterSourceDispose`. **Remainder closed 2026-06-02:** `StaticFrameSource` and `ImageFileSource` now refcount their shared backing too (source ref + one per live frame; the release hook / `ArrayPool.Return` fires only once the source AND all emitted frames are disposed). This became a *real* use-after-free once P2-14 made `StaticFrameSource.FromFrame` always pass `owned.Dispose` (an `ArrayPool`-backed clone) as the release hook, and `ImageFileSource` always pools — disposing either mid-playback returned a buffer a queued frame still aliased. `StaticFrameSource` only refcounts when a release hook is present, so the plain managed-array path stays zero-overhead (GC-safe). Regression tests `StaticFrameSourceTests.Dispose_WithReleaseHook_DefersReleaseUntilInFlightFramesDisposed` / `ReleaseHook_FiresOnce_WhenFrameOutlivesSource_AndFrameDisposeIsIdempotent`.

- [x] **P1-2 Pause order.** **Closed via** pausing audio (router + clock, or the video clock when audio-less) first, then doing the possibly-blocking `video.Pause` in a `finally`, with the shared-mux flush still last.

The video consume below takes `_videoDecodeLock` sequentially, so no nested-lock-ordering hazard.
- [x] **P1-4 Reset fallback PTS counters on seek.** **Closed via** re-anchoring `_vFramesEmitted` (shared demux `SeekPresentation`) and `_framesEmitted` (`VideoFileDecoder.Seek`) to `round(position * fps)` so no-PTS streams resume at ~seek-target instead of a stale future index.
- [x] **P1-5 AudioClipVoice EOF release hang.** **Closed via** finishing the release ramp immediately (`_gain = 0; _stopped = true`) when the source is exhausted mid-release, so `IsExhausted` becomes true instead of lingering. Regression test `AudioClipTests.Voice_StopAtEndOfClip_BecomesExhausted`.
- [x] **P1-6 FFmpeg video encoder allocate-once.** **Closed via** allocating `_frame`'s plane buffers once in `OpenCodecLocked` (format/dims set there), checking the `av_frame_make_writable` return in `Submit`, and removing the per-frame `av_frame_get_buffer` from `FfmpegAvFrameFill.CopyVideoFrame` (now a pure copy with a dimension guard).
- [x] **P1-7 FFmpeg audio encoder non-FLT/FLTP.** **Closed via** a per-encoder `SwrContext` (FLT-in → codec-sample-fmt-out) used in `WriteFrameLocked`, replacing the FLTP-only / raw-float-copy branches. Now correct for FLAC (s16/s32) and any integer/planar codec. Regression test `FFmpegMuxRoundTripTests.Audio_flac_round_trip_preserves_signal` (lossless RMS check).
- [x] **P1-8 Router clock catch-up cap.** **Closed via** re-anchoring the deadline to "now" once the backlog exceeds `MaxCatchupChunks` (64, matching `MediaClock`) in both `WallClockRouterClock` and `PlaybackSlavedRouterClock`.
- [x] **P1-9 Frame-mode audio tail drain.** **Closed in Batch 12.** `AudioFileDecoder.TryReadNextFrame` and shared-demux `AudioTrack.TryReadNextFrame` now drain the `swr` tail into pooled `AudioFrame`s after decoder EOF and set the drained flag once empty, so frame-mode consumers no longer lose tail samples or wait forever for `IsExhausted`.
- [x] **P1-10 ResamplingAudioSource seekability.** **Closed via** a `ResamplingAudioSource.Create` factory that returns a `SeekableResamplingAudioSource : ResamplingAudioSource, ISeekableSource` when the inner is seekable (forwards Seek/Position/Duration and flushes the resampler on seek via `ResetAfterInnerSeek`); the FFmpeg auto-resample factory now uses `Create`. - [x] **P1-11 NDI ingest timing.** **Closed in Batch 12.** `NDIVideoReceiver` and the combined public `NDISource` now preserve the live `RebaseToLatest(playClock.CurrentPosition)` behavior while using NDI-provided timing when available: the first post-rebase timecode/timestamp becomes the session origin and each frame presents at `rebaseBase + (frameNdiTime - origin)`. Streams with synthesized/undefined timing fall back to the existing monotonic synthetic cadence; once format is known, that fallback cadence uses the negotiated frame rate. Unit coverage added for `NDIFrameTiming` origin/timestamp/no-timing behavior; hardware validation is still recommended for real VFR/dropped-frame senders.
- [x] **P1-12 VideoPlayer restart guard.** **Closed via** a `_decodeThreadStuck` flag set when `StopInternal`'s bounded join leaves the decode thread `IsAlive` (and it wasn't an early cooperative cancel); `Play()` then throws instead of starting a second decode thread over the same source/queue.

## Phase P2 — Allocation, contracts, API hardening

- [x] **P2-1 Lazy-start output-pump threads.** **Closed via** deferring the pump's drainer `Thread.Start()` (the OS-thread/stack allocation — the cheap `new Thread(...)` managed object stays in the ctor) behind a new idempotent `OutputPump.EnsureStarted()`, guarded against a late-start-after-dispose race by a `_startGate` lock (a thread started after `_ready`/`_cts` are disposed would crash). The router calls `EnsureStarted()` for every registered pump in `Start()` (before the run loop) and for a single pump in `AddOutput` when already running. A router that's constructed + has outputs added but is never started now spawns **zero** pump OS threads — directly removing the suite-level thread pressure behind the intermittent `OutputPump..ctor` OOM. `Dispose`/`WaitForIdle`/`AbandonQueue` are already safe for an unstarted thread (`JoinThread`/`IsAlive` guard on `IsAlive`, false until started). Regression tests `AudioRouterPumpLifecycleTests.OutputAddedBeforeStart_ReceivesAudio_AfterStart` / `OutputAddedWhileRunning_ReceivesAudio` (both fail if either `EnsureStarted` call site is dropped — audio never drains).
- [x] **P2-2 VideoRouter conversion outside lock.** **Closed (validated by a purpose-built harness).** `VideoRouterInputOutput.Submit` → `InputRegistration.SubmitPhased`: **phase 1** snapshots the routed outputs + their branch converters + fan-out/readback decisions under `_gate` and sets `_convertInFlight` (leasing the converters); **phase 2** does the readback / `Convert` / clone with `_gate` **released** (the expensive branch work no longer blocks route mutations or other inputs' submits); **phase 3** re-takes `_gate`, clears the lease, frees any converters a concurrent reconfigure deferred, and delivers — re-validating each output still exists (it may have been removed during the convert). A per-input `_submitLock` serializes submits on one input (swscale converters aren't reentrant) **without** taking `_gate`, so it can't deadlock against router-level `RemoveOutput`. The use-after-dispose hazard is handled by **deferred converter disposal**: `TearDownPaths` moves a still-leased converter to `_deferredConverters` instead of freeing it; the in-flight submit drains them in phase 3. The single-output path stays a fast under-`_gate` submit. **Harness:** new headless `VideoRouterConcurrencyTests` (injected instrumented converter; submit thread vs route add/remove thread for 600 ms) asserts zero `Convert`-after/over-`Dispose` — **proven to have teeth** (temporarily disabling the lease makes it fail). The 13 existing `VideoRouterTests` (CPU fan-out, branch converters, DRM dma-buf NV12/P010/P016 branches on Linux, async pumps) still pass, plus full Core 417 / FFmpeg 163 / Playback 31. Real-world end-to-end check available via `Tools/VideoPlaybackSmoke --ndi <file>` (SDL3 GL primary + NDI branch converter + PortAudio). **Residual:** the Win32 D3D11 shared-handle branch (Windows-only) is preserved structurally but unexercised on Linux — validate on Windows.
- [x] **P2-3 VideoOutputPump reconfigure drain.** **Closed via** dropping queued frames in `Configure` when the new format differs from the inner's current format (the unchanged-format branch-route re-Configure stays a cheap pass-through). Residual: a frame already in-flight on the drainer can still race — noted; full format-versioned queues left as a follow-up.
- [x] **P2-4 VideoOutputPump dispose vs blocked Submit.** **Closed via** checking `Thread.IsAlive` after the join; if the drainer is still blocked in a slow inner `Submit`, `Dispose` now leaks `_pending`/`_cts`/the inner deliberately (logs) instead of disposing them under the running thread (which would throw `ObjectDisposedException` on the drainer or use-after-dispose the inner).
- [x] **P2-5 Compositor hot-path allocations.** **Closed via** serializing the single-consumer read with a `_readGate` and reusing three per-instance scratch lists (slot snapshot, `CompositorLayer` layers, acquired-slot list) across composites, plus **removing the per-slot `SlotFrameLease` object entirely** — `Slot.AcquireLatest`/`AcquireMasterAligned` now return the `VideoFrame?` directly and register a read-ref, and `TryReadNextFrame` tracks the acquired slots and calls a new `Slot.ReleaseFrame()` in its `finally`. A steady-state composite now allocates only the compositor's output frame (slot-count-independent) instead of a `Slot[]` + two `List<>`s + N leases. Regression test `VideoCompositorSourceTests.SteadyStateCompositeRead_AllocationDoesNotScaleWithSlotCount` (measures per-read bytes at 1 vs 8 slots; fails if per-slot allocation returns). **Left as-is:** the public `Slots` getter still allocates a snapshot array per access — it has no hot-path callers (diagnostics only) and the immutable-snapshot return is the safe contract.
- [~] **P2-6 FFmpeg planar pin/alloc per frame.** **Safe non-escaping reduction done (2026-06-02); the rest remains profiling-gated.** The transient pinning scratch — the `List<GCHandle>` that does NOT escape the method (freed before return) — is now a `stackalloc GCHandle[n]` (n ≤ 4) in both swscale converted paths (`VideoFileDecoder.BuildConvertedPlanarFrame`, `MediaContainerSharedDemux.BuildConvertedPlanarVideoFrame`), with `allocated`-count partial-failure handling so a mid-loop `Rent` failure frees exactly the pinned handles. Removes one `List` + backing-array allocation per converted frame at zero risk (FFmpeg decode/convert suite, 163 tests, green). **Still deferred (per the finding's own "measure first"):** the dominant allocations — `strides` int[], `memories` ReadOnlyMemory[], `buffers` byte[][] — *necessarily escape into the returned `VideoFrame`* (they are its planes/strides and live until it's disposed), so they can't be `stackalloc`'d; pooling them via a per-frame arena returned on dispose (as `PassThroughDescriptorArena` does for the pass-through path) is feasible but adds lifetime-management complexity to the decode hot path for an **unmeasured, marginal** gain (dwarfed by the already-pooled pixel buffers + the sws_scale work). Do it behind a profiler, not blind.
- [~] **P2-7 FFmpeg pass-through per-plane memory-manager alloc.** **Re-verified 2026-06-02 — largely already mitigated.** Both decoders (`VideoFileDecoder.BuildPassThroughFrame` and `MediaContainerSharedDemux.BuildPassThroughVideoFrame`) already pool the per-frame plane/stride **descriptor arrays** via `PassThroughDescriptorArena` (per-plane-count Treiber free-list, `Array.Clear` on return) — the bulk of the original concern. The only residual per-frame allocation is the small `new UnmanagedMemoryManager<byte>(...)` wrapper per plane (n≤4). Pooling those too requires extending the arena handle to carry + reset the wrappers across the frame's lifetime (the wrapper's `.Memory` is handed into the frame, so it can't be reused until the frame is disposed). Low value, profiling-gated — left for if/when a profiler flags pass-through wrapper churn.

## Phase P3 — Architecture (design-level; scope per item)

- [x] **P3-1 Owned MediaSession/PlaybackSession.** **Closed via** a new `MediaSession` (`S.Media.Playback`) — a single owning handle around a `MediaPlayer` plus the resources wired around it. `MediaSession.Owning(player)` disposes the player; `Borrowing(player)` leaves it to the caller. `Own<T>(resource)` registers extra outputs / companion hosts / app disposables (the Borrow escape hatch is simply *not* registering). Disposal is ordered: the player first (so its routers stop before anything still feeding an output is freed — building on P2-18's player-owned companions), then registered resources in reverse order.
