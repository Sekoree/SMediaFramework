# 16 · Type Coverage Appendix

This appendix is the traceability layer for the explained series. The narrative chapters explain the hot paths and design decisions; this file names every production C# type and points it back to the chapter that owns the concept.

Scope: production source under `MediaFramework`, `UI/HaPlay`, and `UI/HaPlay.Desktop`; `bin`, `obj`, and test projects are excluded. The dependency source under `Reference/` is intentionally not cataloged here because it is third-party reference material, not MFPlayer source.

Inventory checked on 2026-06-14: **579 production C# files**, **108,545 production lines**, **1,214 type definitions** across **564 files that declare types**. Top-level-statement smoke tools and global-usings files are covered by their owning narrative docs, but they do not appear in the type table because they declare no source type.

## How to read the tables

* **Doc** is the conceptual chapter to read for that file. For example, `04` means [04 · Core Audio Engine](04-Core-Audio-Engine.md).
* **Types** lists classes, records, interfaces, structs, and enums declared in the file, including private nested helpers when they are explicit source types.
* **Role** comes from the nearest XML summary when one exists; otherwise it is a short generated placement note. If a row says only that it defines types, the owning chapter carries the actual explanation.
